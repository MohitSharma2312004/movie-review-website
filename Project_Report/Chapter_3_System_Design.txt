3.2 DATA NORMALIZATION
Normalization is a process of organizing data in a database to reduce data redundancy and improve data integrity. It is a set of guidelines that helps database designers to create efficient, flexible, and maintainable database schemas.

Normalization 1NF:
First Normal Form (1NF): This level ensures that each table has a primary key and that all columns contain atomic (indivisible) values.

Normalization 2NF:
Second Normal Form (2NF): This level ensures that all non-key attributes are dependent on the primary key and not on other non-key attributes. 
(For the tables outlined in 1NF, they inherently satisfy 2NF as they contain single-column primary keys without partial dependencies)

Normalization 3NF:
Third Normal Form (3NF): This level ensures that all non-key attributes are dependent only on the Primary key and not on other non-key attributes, and that there are no transitive dependencies.

Normalization Level (BCNF):
BCNF stands for Boyce-Codd Normal Form, and it is a higher level of database normalization than the third normal form (3NF). A relation is in BCNF if every determinant is a candidate key.
