Chapter 2
REQUIREMENTS AND ANALYSIS

2.1 Software Requirement Specification
Secure user registration and login system. 
Role-based access for regular users and administrators. 
Integrate movie rating systems out of 5 stars.
Responsive, mobile-friendly UI.
Cloud-based database integration.
RESTful APIs for frontend-backend communication.

2.2 Data Gathering
Conducted surveys with potential movie enthusiasts.
Identified user needs regarding movie search and filtering.
Analyzed existing movie platforms like IMDb and Rotten Tomatoes for feature gaps.
Collected feedback on interface preferences for reading and writing reviews.

2.3 Feasibility Study
Technical Feasibility:
MERN Stack Reliability and Scalability
TMDb or OMDB API Integration (Optional for fetching records)
Cloud Hosting Viability

Economic Feasibility:
Leveraging Open-Source Technologies
Cost-Effective Development
Scalable Hosting Costs

Operational Feasibility:
Intuitive User Interface (UI)
Manageable Admin Interface
Accessibility and Mobile-Friendliness
Local Support and Maintenance

2.4 Hardware Requirements
For Development:
Processor: Intel Core i5 (7th generation or newer) or equivalent AMD Ryzen processor.
RAM: 8 GB minimum, 16 GB recommended.
Storage: 256 GB SSD minimum, 512 GB SSD recommended.
OS: Windows 10/11, macOS (latest supported version) 

For End Users:
Internet-enabled device (PC, tablet, or mobile)
Modern browser (Chrome, Firefox, Edge, Safari)
Reasonable internet speed.

2.5 Software Requirements
Frontend: React.js, Vanilla CSS / Tailwind CSS
Backend: Node.js, Express.js
Database: MongoDB Atlas
Browser Support: Chrome, Firefox, Edge, Safari

2.6 Justification of Selection of Technology
        2.6.1 (USER INTERFACE)
React.js for Dynamic and Responsive UI.
CSS for Rapid and Consistent Styling.
Material UI for Enhanced User Experience and Accessibility.

        2.6.2 (SERVER-SIDE LOGIC)
Node.js and Express.js for High Performance and JavaScript Integration.
RESTful API Design for Clarity and Maintainability.
Efficient Support for Core Reviewing Functionalities.

        2.6.3 (DATABASE)
MongoDB Atlas for Flexible Data Storage and Cloud Convenience.
Efficient Handling of Movie and User Review Data.
Seamless Integration with Node.js via Mongoose.

2.7 Use Case Diagram
A use case diagram is used to represent the dynamic behavior of a system. It encapsulates the system’s functionality by incorporating use cases, actors, and their relationships. It models the tasks, services and functions required by a system/subsystem of an application. It depicts the high-level functionality of a system and also tells how the user handles a system.

2.8 Data Flow Diagram (DFD)
Data Flow Diagrams (DFDs) offer a visual representation of the information flow within the MERN movie review platform. These diagrams provide a structured understanding of how data is processed, its origins and destinations, and where it is stored within the system. For this project, DFDs will model the movement of data between key elements, including external entities like users and administrators, the various processes or functionalities of the platform, and the data stores such as databases. 

2.8.1 Context level DFD – 0 level
DFD Level 0 is also called a Context Diagram. It's a basic overview of the whole system or process being analyzed or modelled. It's designed to be an at-a-glance view, showing the system a single high-level process, with its relationship to external entities.
