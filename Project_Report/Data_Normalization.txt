3.2 Data Normalization

1NF (First Normal Form) - Only Primary Keys
Requirement: Tables only display their Primary Keys.

| Table Name | Primary Key | Attributes |
|---|---|---|
| Users | UserID | Username, Email, Password |
| Movies | MovieID | Title, Genre, ReleaseYear |
| Reviews | ReviewID | Rating, Commentary |


2NF (Second Normal Form) - Primary Keys & Foreign Keys
Requirement: Tables must show Primary Keys and relate via Foreign Keys.

| Table Name | Primary Key | Foreign Key(s) |
|---|---|---|
| Users | UserID | None |
| Movies | MovieID | None |
| Reviews | ReviewID | UserID (Users table)<br>MovieID (Movies table) |


3NF (Third Normal Form) - Only Composite Keys
Requirement: Should only contain Composite Keys (using two or more columns).

| Table Name | Composite Key | Attributes |
|---|---|---|
| Reviews | (UserID, MovieID) | Rating, Commentary |
